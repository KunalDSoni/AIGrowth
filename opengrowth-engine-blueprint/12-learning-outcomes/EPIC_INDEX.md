# Learning and Outcomes Engine

## Mission

Measure what changed after actions were implemented, explain attribution limits, and feed learning back into future recommendations.

## Epics

| Epic ID | Epic | Outcome |
|---|---|---|
| LEARN-001 | Metric Snapshot Contract | Normalize search, analytics, AI visibility, conversion, and campaign metrics |
| LEARN-002 | Baseline Window Builder | Capture pre-implementation metrics and sample-size context |
| LEARN-003 | Implementation Annotation | Record when, what, and how a recommendation was implemented |
| LEARN-004 | Comparison Window Analysis | Compare relevant metrics after implementation |
| LEARN-005 | External Event Annotation | Record seasonality, site changes, campaigns, algorithm updates, and outages |
| LEARN-006 | Outcome Classification | Classify positive signal, negative signal, no change, insufficient data, confounded, or awaiting data |
| LEARN-007 | Attribution Limitation Explainer | Explain why an outcome is or is not confidently attributable |
| LEARN-008 | Learning Record Model | Store what was learned and what should change next |
| LEARN-009 | Reprioritization Feedback | Feed outcomes into recommendation scoring |
| LEARN-010 | Outcome UI | Show baseline, comparison, confidence, evidence, and next action |

## First Vertical Slice

Start with the smallest slice that creates evidence, produces a decision, and gives the user a meaningful next action.
