# 90-Day Engine Roadmap

This roadmap is designed for a founder using Codex to build a credible industry-grade demo. It prioritizes real intelligence over broad product surface area.

## Day 1-10: Evidence-Backed SEO Decision Loop

Goal:

```text
Business profile
→ page evidence
→ technical observation
→ recommendation
→ score explanation
→ user status
```

Deliverables:

- Capability registry
- Evidence model
- Northstar demo business
- One normalized page evidence record
- Metadata/indexability audit rule
- Recommendation candidate
- Transparent score
- Evidence drawer
- Tests

Do not build:

- Marketing site redesign
- Billing
- Broad navigation
- Generic AI chat

## Day 11-20: Business Graph and Site Inventory

Goal:

```text
Services + audiences + goals
→ page classification
→ content coverage gaps
```

Deliverables:

- Business knowledge graph
- Service/product catalogue
- Audience segments
- Geography and conversion goals
- Site inventory
- Page purpose classification
- Missing service/audience page gap

## Day 21-30: Search and Prompt Opportunity Engine

Goal:

```text
Business context
→ topic/prompt opportunities
→ intent/funnel classification
→ ranked opportunity
```

Deliverables:

- Demo search/prompt provider
- Prompt/topic demand proxy
- Intent classifier
- Topic clustering
- Funnel-stage mapping
- Opportunity score
- Recommendation candidate

## Day 31-45: AI Visibility Observation Engine

Goal:

```text
Prompt family
→ variants
→ AI observations
→ mentions/citations
→ sample size/variance
```

Deliverables:

- Prompt family model
- Prompt variant model
- Mock AI visibility provider
- Observation run lifecycle
- Raw answer storage
- Mention extraction
- Citation extraction
- Sentiment placeholder
- Variance and sample-size metrics
- Evidence UI

## Day 46-55: Citation Intelligence

Goal:

```text
AI citations
→ source domains
→ competitor/user gap
→ content/source action
```

Deliverables:

- Citation normalization
- First-party vs competitor vs third-party classification
- Cited page/domain aggregation
- Citation gap detector
- Sample-size threshold
- Citation-to-brief recommendation

## Day 56-65: Content Inventory and Refresh Intelligence

Goal:

```text
Inventory + performance + content quality
→ refresh action
```

Deliverables:

- GSC-ready content inventory
- Target query mapping
- Simulated performance metrics
- Content decay signals
- Trust/proof analysis
- CTA clarity analysis
- Internal link opportunity signals

## Day 66-75: Evidence-Grounded Brief and Asset Generation

Goal:

```text
Recommendation
→ brief
→ draft
→ claim flags
→ approval
```

Deliverables:

- Brief builder
- Brief templates by content type
- Mock generation provider
- Metadata generation
- Service page outline
- Claim verification flags
- Version history
- Approval state

## Day 76-85: Outcome Learning Loop

Goal:

```text
Completed action
→ baseline/comparison
→ outcome signal
→ learning
```

Deliverables:

- Metric snapshot model
- Baseline window
- Implementation annotation
- Comparison window
- Outcome classification
- Attribution caveats
- Learning record

## Day 86-90: Demo Hardening

Goal:

Make the demo credible to agencies, founders, and technical reviewers.

Deliverables:

- Polished Today view
- Recommendation detail
- Evidence drawer
- AI visibility view
- Citation view
- Content brief view
- Outcome view
- Demo script
- Known limitations
- Tests passing
- Build passing

## Demo Story

The demo must show:

```text
Northstar Accounting wants more qualified consultation bookings.
OpenGrowth understands services, audiences, and goals.
It inspects website evidence.
It finds a missing medical-clinic bookkeeping opportunity.
It also sees competitors appearing in AI answers and citations.
It explains evidence and uncertainty.
It recommends a specific action.
It generates a brief and metadata/page draft.
The user approves.
The system defines measurement.
Later, it classifies the outcome with caveats.
```
